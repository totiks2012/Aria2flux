# Aria2flux
This project is a high-tech TUI shell over the powerful aria2c engine, designed to provide maximum comfort and speed when working with downloads in the terminal.

Этот проект представляет собой высокотехнологичную TUI-оболочку над мощным движком `aria2c`, разработанную для обеспечения максимального комфорта и скорости при работе с загрузками в терминале.

# ARIA2 FLUX (v6.2 aria2c Shell) 🚀

**Aria2 Flux** — это реактивный консольный менеджер загрузок, написанный на **Go**, который превращает работу с `aria2c` в интуитивно понятный и мгновенный процесс. Проект построен вокруг идеи «Deep Static» — интерфейс должен реагировать быстрее, чем сеть.

## 🛠 Какую «боль» решает этот проект?

Использование `aria2c` в чистом виде или через стандартные Web-интерфейсы часто сопряжено с рядом неудобств, которые **Aria2 Flux** полностью устраняет:

* **Слепота демона:** Чистый `aria2c` в режиме демона требует постоянных запросов через терминал для проверки статуса; Flux дает живой дашборд с частотой обновления 60 FPS.
* **«Магнитная амнезия»:** При добавлении Magnet-ссылок Aria2 часто не показывает имя файла, пока не скачает метаданные; Flux использует **Shadow Cache**, чтобы мгновенно отобразить название из вашей истории.
* **Визуальный мусор:** Сетевые протоколы генерируют много технических задач (например, `[METADATA]`); наша  оболочка фильтрует этот шум, оставляя только важные файлы.
* **Сложность настройки:** Вместо ручного прописывания десятков флагов RPC и путей, Flux берет на себя всё управление жизненным циклом сессии.

## 🏗 Архитектура системы

Проект разделен на две логические части, работающие в связке:

1. **Скрипт-враппер (`l_flux.sh`):** «Инженерный отсек» системы. Он управляет запуском демона `aria2c`, проверяет доступность RPC-порта, следит за сохранением сессии и компилирует бинарный файл при необходимости.
2. **Бинарный файл (`flux2`):** «Лицо» системы. Высокопроизводительный интерфейс на Go, который общается с демоном через JSON-RPC и обеспечивает мгновенную отрисовку списка задач.

---

## 📋 Зависимости

Для корректной работы системы в вашей ОС (рекомендуется MiniOS или любой современный Linux) должны быть установлены:

* **`go`**: Для компиляции бинарного файла `flux2`.
* **`aria2c`**: Основной движок загрузок.
* **`fzf`**: Используется для выбора директорий в конфигурационном скрипте.
* **`micro`** (или `nano`): Для быстрого редактирования списка ссылок.

## 🚀 Как запустить

Вам не нужно запускать бинарный файл напрямую, так как без активного демона он не сможет получить данные. Используйте скрипт-враппер:

```bash
# Перейдите в директорию проекта и дайте права на выполнение
chmod +x l_flux.sh

# Запустите оболочку
./l_flux.sh

```

**Что произойдет при запуске:**

1. Скрипт проверит наличие бинарного файла `flux2` и соберет его, если его нет.
2. Запустит `aria2c` в фоновом режиме с оптимизированными настройками для DHT и Peer Exchange.
3. Дождется открытия RPC-порта и передаст управление интерфейсу.
4. При выходе (F10) скрипт корректно сохранит сессию и завершит работу демона.

## ⌨️ Управление (Hotkeys)

| Клавиша | Действие |
| --- | --- |
| **ENTER** | Добавить новую прямую ссылку или Magnet-ссылку. | то есть эта программа умеет скачивать как торрент - магнет файлы так и обычные
| **TAB** | Циклическое переключение между папками загрузки из `config.json`. | перед запуском создайте в каталоге от запускаемого бинарника файл `config.json` и пропишите в нём пути каталогов для выбора скачивания файлов например 

```
{
  "allowed_folders": ["/home/live/Downloads" , "/home/live/TOR" , "/home/live/ISO/"]
}

```

| **F4** | Открыть `downloads.list` в редакторе для массового добавления ссылок. |
| **F5** | Удалить выбранную задачу и её результат из памяти Aria2. |
| **UP / DOWN** | Навигация по списку закачек. |
| **F10** | Безопасный выход из программы. |

## ⚙️ Технологический стек

* **Language:** Go (Golang).
* **TUI Framework:** Bubble Tea (Charm Cloud) — обеспечивает событийную модель интерфейса.
* **Styling:** Lip Gloss — отвечает за фосфорные цвета и адаптивную верстку.
* **RPC:** Argo — легковесный клиент для общения с API Aria2.

---

> **Инженерная заметка:** Благодаря использованию `strings.Builder` и Snapshot-кэширования, нагрузка на CPU при отрисовке интерфейса стремится к нулю, что делает **Aria2 Flux** идеальным инструментом для слабых машин и серверов.
